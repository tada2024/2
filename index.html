絶対パスと相対パス

パス=Excelのシートやマクロから別のブックを参照するとき、HTMLのタグやタグでリンク先や画像などのファイルを指定するときに使う

  • 絶対パス=目的地における住所を記載する。例→img src=http://@@@.co.jp/img/dpg.jpg　各階層をスラッシュで区切る(windowsではたまに¥も使ったり)
  • 相対パス=目的地における現在位置からの道順を記載する。例→img src=../dog.jpg　一つ上の階層では../　二つ上では../../　同階層では./を使う
  • 一つ上の階層にあり、別のファイルに入っているとき→../jpeg/dog.jpg　といったように別途フォルダも指定する。
それぞれのメリットデメリット
絶対パス　メリット
  1. 目的ファイルの位置が変わらなければ記述も変わらない。→htmlファイルを別の場所に移動させてもパスはいじらなくてよい
  2. 外部サイトへのリンク指定においては必須
絶対パス　デメリット
  1. パスが長くなる
  2. 収容するサーバやドメイン名が変わったときにはすべての記載を変更しなくてはならない
相対パス　メリット
  • サイトの引っ越しなどでサーバーが変わっても、ファイル同士の位置が変わらなければ記載の変更の必要なし
  • パスが短い
相対パス　デメリット
  • ファイルを別のところへ移動させる場合、パスの書き換えが必要になる
ルートパスについて→Webサイトからのルートを記載する

例→ img sec="/img/dog.jpg"　これはフォルダの階層関係なく目的フォルダを指定できるため、ドメイン変更を伴うサーバー引っ越しの時でもリンクを書き換えなくてよい

使用頻度の高い疑似クラス
  1. :link=未訪問リンクのCSSスタイルを指定
  2. :visited=訪問済みリンクのCSSスタイルを指定
  3. :hover=要素にマウスカーソルが重なった時のCSSスタイルの指定
  4. :active=要素をクリックして離すまでの間のCSSスタイルを指定
    上記の要素は:link → :visited → :hover → :activeなど記載順に気を付けないと要素が上書きされてしまうかもしれない visited→hoverの順だとvisitedの要素が反映されない
  5. :focus=フォーカスされた要素のCSSスタイルを指定
  6. :first-child=ある要素で一番最初に記載された要素にのみ指定する
  7. :last-child=ある要素で最後に記述された要素にのみ指定する
  8. nth-child=ある要素でN番目に記述された要素にのみ指定する(n=4,2n+1,even偶数.pdd奇数も可)
  9. nth-last-child=ある要素で最後からN番目を指定

尚、childを使った疑似クラスは一番最初の要素であるなら別の要素であっても参照してしまうので、
そういった場合は:first-of-typeやlast-of-typeなどを使うとよい

  2. ユニバーサルセレクタ=すべての要素を対象にCSSを適用する。セレクタに*をつけて指定

CSSが適用される優先順位

CSSは、同じ要素に異なるCSSスタイルを何度でも上書きできる。よって、ヴラウザがどのスタイルを適用するか判断するための優先順位がある

基本的には後から読み込まれたスタイルが優先的に適用される。が、記載場所と詳細度によって変動もする

記載場所は、インライン、ヘッダー、外部参照の3パターンあり、インラインが最も優先されるヘッダーと外部は同等

CSSはセレクタの書き方によって、詳細度という概念で評価され、優先順位を決定する。

id class タグ
詳細度 1.0.0 0.1.0 0.0.1

最新のバージョン番号が付いたセレクタが優先適用される(id→class→タグ)

!important指定

強制的にスタイルを適用させる。color:red !important;　といったように表記

多用しすぎると優先順位のルールが崩れて、メンテがむずくなるので注意。
尚、HTMLメールを作成する場合は、メーラー固有のCSSスタイルを打ち消す目的で、意図的に「!important指定」を多用する場合もある

フレックスコンテナに設定できる項目

  • 親要素に高さやpaddingを指定していない

デメリット

  • サイトの読み込み速度が遅くなる

transitionについて

transition→変化を見せるプロパティ。CSSプロパティの上書き前と上書き後のスタイルの変化を、指定した時間かけて見せるプロパティ。

プロパティ一覧

  • transition-duration＝変化の秒数
  • transition-delay=変化が開始するまでの秒数
  • transition-property=変化させたいプロパティ
  • transition-timing-function=変化の進行度

複数のプロパティを選択するときはtranslate: a, b;やtranslate: all;と表記する

transition-timing-functionについて

アニメーションの進行度度合いを設定できる(6種)

  • ease=開始時と終了時の動きを滑らかにする
  • ease-in=開始時だけ滑らかにする
  • ease-out=終了時のみ滑らかにする
  • ease-in-out=easeよりもゆっくり変化させる
  • linear=一定の速度で変化させる
  • cubic-bezier=変化の度合いをX軸、Y軸の曲線で指定できる

transition: プロパティ 秒数 変化を遅らせる 進行度合い;
transition: all 1s 0s ease;みたいな一括指定もできる。durationが先になる

transitionとCSSアニメーションとの違い

transitionでは簡単な設定、CSSアニメーションでは高度な設定が可能。CSSアニメーションを使うと、変化を秒数によって細かく刻んだりループさせたりなどの高度な設定ができます。transitionではここまで高度な設定はできませんがコードの記述が簡単というメリットがあります。

CSSアニメーションでは@keyframesの設定が必要。CSSアニメーションを使うときは、@keyframesという設定が必要となります。0%〜100%の数値で開始時・中間地点・終了時の任意のタイミングで変化を切り替えるポイントを設定できる方法です。本格的なアニメーションを作成したいときに使用します。

transitionでは要素を変化させるための擬似クラスが必要。transitionを使用するときには要素の状態を変化させるため、ホバー時（要素にカーソルを合わせたとき）やクリック時（要素をクリックしてから離すまで）などの状態を変化させる擬似クラスを使用する必要があります。

transitionを使うことが多い場面

  1. ホバー時
  2. クラスが追加・解除されたとき。JavaScriptで動きをつけてtransitionでアニメーションの肉付けをする、という流れでclassの追加や解除を行ったときにもtransitionを使うことが多いです。

transitionが効かないとき

displayプロパティにtransitionを設定している→transitionプロパティはdisplayプロパティには作用されない

疑似クラスにtransitionの設定をしている→:hoverや:activeなどに指定しても動作しない

animationと@keyframesの指定方法

@keyframesを使うと、transitionではできなかった高度なアニメーションが実装できる。

animationではアニメーションの秒数や回数を指定し、@keyframesではアニメーションがどのように変化するかを指定できる

keyframesは0~100%,to~fromの二種類で指定できる

@keyframes animation{　←記述例
0%(from){
/*アニメーションを開始するときのCSSを記述*/
}

50%{
/*アニメーションの中間のCSSを記述*/
}

100%(to){
/*アニメーションを終了するときのCSSを記述*/
}
}

animationの指定方法

  1. animation-name=アニメーションに名前を付ける。実装するときは必ず名前を定義する必要がある。これを@keyframesでいじる
  2. animation-duration=アニメーションの秒数を指定するプロパティ。初期値は0s、指定できる単位はsかms。
  3. animation-delay=文字通り遅延をかける初期値、指定できる単位は同上
  4. animation-timing-function=アニメーションの進行度合いを設定する。transition-timing-functionで指定できる値にプラスして、steps(コマ送りのように変化させる)がある。
  5. animation-fill-mode=アニメーション終了時の状態を指定するプロパティ。none,forwards,backwards,bothにてそれぞれ指定なし、終了時の状態を維持する、開始時の状態に戻る、開始時にforwards、終了時にbackwardsを適用する
  6. animation-iteration-count=アニメーションがループする回数を指定するプロパティ。数値を指定するとその数だけアニメーションがループする。infiniteで無限
  7. animation-direction=アニメーションの向きを指定するプロパティ。normal,reverse,alternate,alternate-reverseでそれぞれ通常通り、逆、奇数通常偶数逆、奇数逆偶数通常の挙動をとる。
  8. animationでまとめて指定もできる→animation: animation forwards 3s infinite ease 5s alternate;
    それぞれ名前、終了時の状態、アニメーションの秒数、ループ回数、進行度、開始までの秒数、アニメーションの向き。duration,delayはdurationが先

モーダルウィンドウ

概要

画面を押すとその上に新たなウィンドウ画面を表示する機能のこと。
ポップアップウィンドウとは違い、その画面での操作を終えないと画面を遷移することができない

作り方は見た目の作成→その動きを切り替える処理の作成といった手順を用いる。これは多くのパーツで適用できる考え方

タブ切り替え

詳しくはダウンロードしたページへ。仕組みとしては

  • jQueryでタグをつけたり外したりするコードを書く(removeclass,addclass)
